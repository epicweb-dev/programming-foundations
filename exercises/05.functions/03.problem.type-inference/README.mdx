# Type Inference

<EpicVideo url="https://www.epicweb.dev/tutorials/programming-foundations/5-functions/type-inference" />

👨‍💼 TypeScript is smart about inferring types. You don't always need to write
them explicitly—especially for return types.

```ts
function add(a: number, b: number) {
	return a + b // TypeScript knows this returns number
}
```

But TypeScript **cannot** infer parameter types (in most cases). You must
specify them:

```ts
// ❌ Error: Parameter 'a' implicitly has 'any' type
function add(a, b) {
	return a + b
}

// ✅ Correct
function add(a: number, b: number) {
	return a + b
}
```

3. Fix `divide` so dividing by zero throws an `Error` with the exact message

4. Create `isEven(n: number)` that returns whether `n` is even; let TypeScript

5. Export `multiply`, `divide`, and `isEven`

## Required exports

`multiply`, `divide`, `isEven`

## Completion criteria

- `multiply(4, 5)` → `20` (and other products work)
- `divide(10, 2)` → `5`; `divide(7, 2)` → `3.5`
- `divide(10, 0)` throws with message `"Cannot divide by zero"`
- `isEven(4)` / `isEven(0)` → `true`; `isEven(7)` → `false`
