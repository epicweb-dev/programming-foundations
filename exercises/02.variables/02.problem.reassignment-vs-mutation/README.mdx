# Reassignment vs Mutation

ğŸ‘¨â€ğŸ’¼ There's a subtle but important distinction between **reassignment** and
**mutation**. Understanding this prevents bugs that are hard to track down.

**Reassignment**: Pointing a variable to a completely new value

```ts
let x = 1
x = 2 // x now points to a new value
```

**Mutation**: Modifying the contents of an existing value

```ts
const arr = [1, 2, 3]
arr.push(4) // The array itself is modified (mutated)
```

<callout-warning>
	`const` prevents reassignment but NOT mutation! You can still push to a
	`const` array or add properties to a `const` object.
</callout-warning>
