# Throwing and Catching Errors

<EpicVideo url="https://www.epicweb.dev/tutorials/programming-foundations/4-control-flow/errors" />

👨‍💼 We need to convert user input into a number. If the input is invalid, we
should throw an error and handle it gracefully so the program can keep running.

Sometimes the best control flow is to stop execution and report a problem:

```ts
try {
	const parsedValue = Number('not-a-number')
	if (Number.isNaN(parsedValue)) {
		throw new Error('Something went wrong')
	}
} catch (error) {
	console.error('Caught an error:', error)
}
```

## Completion criteria (with this fixture)

- `hadError` is `true`
- `resultMessage` is exactly `Error: Invalid number: not-a-number`

Edge cases to keep in mind: valid numeric strings should succeed and set
`resultMessage` to the success format (no error); invalid input must throw and
be caught without crashing the program.

<callout-info>
	In a `catch` block, the caught value is typed as `unknown`. Narrow it before
	you read properties like a message string.
</callout-info>
